# Architecture Rules (Non-Obvious Only)

- **Two-step bootstrap**: You MUST apply [`appprojects-app.yaml`](bootstrap/dev/appprojects-app.yaml) (wave 0) FIRST, wait for it to sync, then apply [`root-app.yaml`](bootstrap/dev/root-app.yaml). Order matters because AppProjects must exist before apps referencing them can be created.
- **AppProject vs Application namespace**: AppProjects and Applications all live in `namespace: argocd`, but the resources they deploy go to various namespaces (kube-system, infrastructure, traefik, monitoring, workload). The infrastructure ApplicationSet uses `CreateNamespace=true` + `ServerSideApply=true` to handle this.
- **Sealed secrets → Reflector chain**: sealed-secrets controller decrypts SealedSecrets in `kube-system`, then Reflector ([`apps/infrastructure/reflector/`](apps/infrastructure/reflector/)) copies annotated secrets to target namespaces. This is why infra-secrets is wave 3 and reflector is wave 4 — reflector must deploy after the secrets it mirrors exist.
- **Workload multi-source pattern**: Each workload Application uses TWO sources: the Helm chart repo (for the chart/template) and a `$values` ref to this repo (for `values.yaml` files). The [`templatePatch`](appsets/dev/workload-appset.yaml:50) builds this via go templating on `config.json` fields.
- **`selfHeal: false` in prod** is a deliberate safety measure — it prevents ArgoCD from auto-reverting manual production changes (e.g., scaling, emergency config changes). Manual syncs still work.
- **The `config.json` `isGitRepo` flag** controls whether ArgoCD treats the source as a Helm chart (`chart:`) or a Git directory (`path:`). This means the same workload appset pattern supports both Helm OCI charts and raw Git-hosted manifests.
- **Port forwarding in k3d**: The [`k3d-dev.yaml`](.devcontainer/manifests/k3d-dev.yaml:8) maps specific host ports (30179, 30280, etc.) to cluster NodePorts. Adding a new service requires both a k3d port mapping AND a `forwardPorts` entry in [`devcontainer.json`](.devcontainer/devcontainer.json:25).
